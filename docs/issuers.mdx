# Issuers: receive a receipt

The Receiver is a protocol client typically operated by a card issuer or other consumer of receipt data. Their responsibilities are:

1. listening for receipt data on the receiver addresses registered with the Registry
2. exchanging transaction hashes for decryption keys upon receiving data
3. decrypting and returning receipt data upon customer request
